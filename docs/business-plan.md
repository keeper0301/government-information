# 정책알리미 사업계획 (v0.1)

## Context

**왜 이 문서가 필요한가:** 프로젝트 "정책알리미"는 현재 기술적으로는 Phase 1~3이 구현되어 복지·대출·맞춤추천·AI상담·알림센터 등 핵심 기능이 동작하는 상태. 소셜 로그인도 방금 추가됨. 그러나 **사업계획 문서는 없음** — "누구에게, 무엇을, 어떻게 팔 것인가"가 합의되지 않은 상태. 이 문서가 그 합의서.

**목표:** 6개월 내 수익 발생. 월 9,900원 유료 구독 + 애드센스 광고 하이브리드 모델로 시작.

**이 문서의 범위:**
- 포함: 비즈니스 모델, 타겟, 수익 구조, 6개월 로드맵, 핵심 지표, 사전 준비 리스크
- 미포함: 코드 구현 세부 (Phase 1~3 기술 계획서 참고), 디자인 가이드 (design spec 참고)

---

## 1. 제품 한 줄 정의

> **"놓치고 있던 정부 지원금·복지·대출을 자동으로 찾아 카카오톡·이메일로 알려주는 구독 서비스"**

- **핵심 약속 1개:** "당신이 받을 수 있는 지원금을 놓치지 않게 해준다."
- **경쟁 대상:** 복지로, 청약홈, 소상공인24, 정부24 (모두 **분산된 공식 사이트**)
- **차별점:** ① 통합 검색·필터 ② 개인 조건 기반 자동 매칭 ③ **능동적 푸시 알림**(기존 사이트는 사용자가 직접 찾아가야 함)

## 2. 타겟 사용자

**1차 의사결정: 전 국민 대상 (사용자 선택)**

- 사용자의 공식 포지셔닝은 "세그먼트를 좁히지 않음 — 누구나 쓸 수 있게"
- 카테고리: 청년·자영업자·육아가구·노년층 전 계층 커버

**⚠️ 사업계획 관점에서의 권고사항 (실행 시 반영 필요):**

"전 국민 대상"은 브랜드 메시지로는 훌륭하지만, **마케팅 실행 단계에서는 한 세그먼트를 먼저 공략**하는 게 훨씬 효율적임. 이유:
- 광고 키워드·SEO 키워드가 세그먼트별로 갈림 ("청년월세지원" vs "소상공인대출" vs "출산장려금")
- 검색 유입·커뮤니티 마케팅 타겟이 다름
- 초기 팬 확보에 필요한 "딱 맞는 사람 1천 명"이 흩어짐

→ **실행 단계 추천:** 브랜드는 "전 국민"을 말하되, 첫 3개월 마케팅 집중점은 **1개 세그먼트 선택**. 후보:
1. **청년 (20-30대)** — 검색량 가장 크고 디지털 친숙, 월세·취업지원 키워드 많음
2. **소상공인** — 구독 전환율 높을 가능성 (사업 관련 정보에 지불 의사 큼)
3. **육아 부모** — 커뮤니티 기반 입소문 효과 강함, 네이버 카페 활성

**이 선택은 실행 단계에서 다시 논의.** 이 문서에서는 "전 국민" 포지셔닝을 유지.

## 3. 수익 모델

### 3.1 구조 (하이브리드)

| 구분 | 무료 사용자 | 유료 구독자 (월 9,900원) |
|---|---|---|
| 웹사이트 탐색 | ✅ 전체 공개 | ✅ |
| 필터·검색 | ✅ | ✅ |
| 마감 알림 (이메일) | ⚠️ 주 1회 요약 | ✅ 즉시·개별 알림 |
| **카카오톡 알림톡** | ❌ | ✅ **핵심 유료 기능** |
| 맞춤 브리핑 (AI 요약) | ❌ | ✅ |
| 애드센스 광고 | ✅ 노출 | ❌ 제거 |

### 3.2 단가 근거 (간이 경제성)

- **가격:** 월 9,900원 (부가세 포함 시 실수령 약 9,000원)
- **카카오 알림톡 원가:** 건당 약 8~15원 (카카오 비즈니스 공식 단가)
  - 사용자 1명당 월 평균 20건 알림 = 원가 약 200~300원
- **서버·DB 원가:** Supabase·Vercel·Resend 기본 요금제로 월 1,000명까지 약 5만 원 이하
- **결제 수수료 (토스페이):** 3.3% 내외 → 약 330원
- **사용자당 순이익 추정:** 약 9,000원 - 500원 (알림+인프라) - 330원 (PG) = **약 8,100원**

### 3.3 애드센스 (보조 수익)

- 무료 사용자 페이지에만 노출
- **대략적 기대치:** 한국 CPM 낮음. 월 MAU 10,000명 × 페이지뷰 3회 × CPM 2,000원 = **약 6만 원/월** (참고용, 실측 필수)
- 주 수익원이 아닌 "서버비 보조" 수준으로 포지셔닝

## 4. 6개월 로드맵 & 핵심 지표

### 4.1 목표 (사용자 선택: "수익 발생 시작")

**M6 목표:** **유료 구독자 50명 돌파** (월 매출 약 495,000원)
- 이 숫자는 "수익 발생 시작"의 최소 증명 수준
- 50명 = 전환율 1% 가정 시 무료 회원 5,000명 필요
- 무료 회원 5,000명 = 월 방문자 25,000명 (가입 전환 20%)

### 4.2 월별 마일스톤

| 월 | 핵심 활동 | 성공 지표 |
|---|---|---|
| M1 | **사전 준비 & 인프라** (§6 리스크 해소) | 사업자등록·카카오비즈·토스페이 승인 |
| M2 | 유료 기능 구현 (카톡 알림톡 연동, 토스페이 결제) | 결제 플로우 동작 |
| M3 | **SEO 콘텐츠 생산** (블로그 20편) | 구글 검색 색인, 주간 UV 500 |
| M4 | 커뮤니티 마케팅 (네이버 카페·오픈채팅) | 무료 가입자 1,000명 |
| M5 | 전환 최적화 (무료→유료 유도 UX) | 유료 20명 |
| M6 | 반복·확장 | **유료 50명, 월 매출 50만 원** |

### 4.3 북극성 지표 (단 하나)

- **유료 구독자 수**
- 보조 지표: (1) 유료 전환율 (2) 월 이탈률 (3) 알림 열어본 비율

## 6. 사전 준비 리스크 (⚠️ 중요)

**M1에 반드시 해결해야 유료 기능 구현 가능. 놓치면 M2~M6 일정 전체가 밀림.**

### 6.1 사업자등록

- **토스페이·카카오 알림톡 모두 사업자등록 필수** (개인 신분으로는 불가)
- 홈택스 온라인 신청, 약 1~3일 소요
- 통신판매업 신고 추가 필요 (구독 서비스는 통신판매에 해당)
- **비용:** 통신판매업 등록면허세 약 4만 원/년

### 6.2 카카오 알림톡 연동

- 카카오 비즈니스 채널 개설 → 발신 프로필 등록 → 템플릿 사전 심사
- 심사 기간: **평균 2~3주** (광고성 단어 들어가면 반려)
- 중개사 이용 시 월 기본료 + 건당 과금 구조
- 후보 중개사: 알리고, NHN클라우드, 비즈엠 등

### 6.3 토스페이 정기결제 (CMS)

- 토스페이먼츠 가맹점 신청 (사업자등록증 + 통신판매업 + 홈페이지 검수)
- 정기결제는 "빌링키 발급" API 필요 → 기술 구현 약 1주
- **심사 기간: 약 2주**
- 수수료: 3.3% 내외

### 6.4 개인정보·법률

- **개인정보처리방침 필수** (이메일·카톡 수집 시)
- 전자상거래법상 "전자적 증빙" 보관 (결제 기록 5년)
- 약관 마련 (구독 해지·환불 정책)
- → 표준약관 활용해도 되지만 변호사 검토 1회 권장 (약 20~50만 원)

## 7. 기술 실행 우선순위

이 문서는 사업계획이므로 코드 레벨 실행은 별도. 다만 사업목표에서 역산한 **구현 우선순위**:

1. **결제 플로우** (토스페이 빌링키 → 구독 상태 관리 → 해지)
2. **카카오 알림톡 송신 파이프라인** (큐 + 재시도 + 실패 로깅)
3. **맞춤 매칭 로직 고도화** (현재 단순 필터 → 사용자 조건-프로그램 매칭 정확도)
4. **SEO 최적화** (이미 Phase 3에 ISR 있음 — 블로그 콘텐츠 구조 추가)
5. **전환율 측정 툴** (GA4 이벤트 + 무료→유료 퍼널 트래킹)

각 항목은 이 계획 승인 후 별도 아키텍처 리뷰 권장.

## 8. 검증 방법 (이 사업계획이 맞는지 확인하는 법)

- **M1 종료 시:** 카카오·토스페이·사업자 3종 승인 여부 = Yes/No 바이너리. No 면 일정 전체 조정.
- **M3 종료 시:** 주간 UV 500 달성 여부. 미달 시 콘텐츠 전략 재수립.
- **M6 종료 시:** 유료 50명 도달 여부. 미달 원인 분석:
  - (A) 가입까지는 오는데 유료 전환 안 됨 → 가격·가치 제안 재설계
  - (B) 가입 자체가 적음 → 타겟 세그먼트 집중 (§2의 권고 참고)
- **사용자 인터뷰:** M3·M5 각 5명씩 유료 전환자와 통화 10분 (왜 결제했는지 / 가장 유용한 기능 / 해지 고려 이유)
