# keepioo 앱스토어 출시 계획서 (Capacitor 하이브리드)

작성: 2026-06-15 (office-hours 세션)
대상: keepioo · 정책알리미 (Next.js 16 + Supabase, 이미 PWA 가동 중)
운영: 비개발자 1인

---

## 1. 목적 (사장님 선택)

- **① 앱스토어 노출·신뢰도** — 사람들이 스토어에서 "복지·정책" 검색 시 노출 + "앱이 있다=믿을 만하다" 인상. **(주 목적)**
- **② 푸시 알림 재방문** — ⚠️ 이미 PWA 푸시로 됨(iOS 16.4+도 지원). 네이티브 앱은 푸시가 더 안정적인 정도의 보너스.
- **④ 인앱 결제·수익** — ⚠️ 1차 출시에서 **제외**. 이유는 아래 4번.

사용자 요청(③)은 아직 없음 = 수요는 "있으면 좋다" 수준이지 "달라고 아우성"은 아님. 과투자 경계.

## 2. 선택 경로: Capacitor 하이브리드

기존 웹(Next.js)을 그대로 감싸되, 애플 심사를 통과할 만큼의 네이티브 기능만 얹는 방식.
- 처음부터 다시 안 만듦 (React Native 재작성=비현실적, 기존 자산 버림 → 기각)
- 단순 웹 래퍼(TWA만)보다 애플 거부 리스크 ↓

## 3. 1차 출시 범위 (작게 시작)

**넣는 것**: 정책·복지·대출 정보 열람, 검색, 북마크, 알림(푸시), 공유.
**빼는 것**: 인앱 결제/구독. 결제는 당분간 **웹(토스)에서** 하도록 유도하거나 1차엔 노출 최소화.

## 4. 핵심 리스크 (정직하게)

- **애플 가이드라인 4.2 (minimum functionality)**: "웹사이트를 그냥 복사한 앱"은 거부. → Capacitor로 **네이티브 기능(네이티브 공유 시트, 네이티브 푸시, 딥링크, 오프라인 캐시)** 을 실제로 붙여 "앱다움"을 확보해야 통과.
- **인앱 결제 충돌**: 앱 안에서 구독 결제를 받으면 애플·구글이 자사 결제(수수료 최대 30%) 강제. 토스 결제를 그대로 넣으면 애플이 "외부결제 우회"로 거부 위험. 한국은 외부결제 허용 법이 있으나 애플 심사·수수료는 여전히 복잡 → **1차 제외가 안전**.
- **비개발자 한계**: 빌드·코드는 클로드가 하지만, **개발자 계정 가입·심사 제출·본인 신원/결제 입력은 사장님 본인만** 가능(클로드가 대신 못 함, 안전 원칙).

## 5. 단계별 계획 (누가 무엇을)

| 단계 | 내용 | 담당 |
|---|---|---|
| 1 | Capacitor 설치 + 기존 웹 연결 + 안드로이드/iOS 프로젝트 생성 | 클로드(코드) |
| 2 | 네이티브 기능 추가 — 공유 시트, 네이티브 푸시 연동, 딥링크, 오프라인 폴백 (애플 4.2 대응) | 클로드(코드) |
| 3 | 앱 아이콘·스플래시·스토어 스크린샷·설명문 준비 (개인정보처리방침은 이미 있음=/privacy) | 클로드(자산 생성) + 사장님(확인) |
| 4 | **Apple Developer 계정($99/년)** + **Google Play 계정($25 1회)** 가입 | 사장님 본인 |
| 5 | 스토어 등록 정보 입력 + 빌드 업로드 + 심사 제출 | 사장님(클로드가 단계별 가이드) |
| 6 | 심사 대응(거부 시 사유별 보완) | 클로드 + 사장님 |

## 6. 비용·시간 추정 (1인)

- 비용: Apple $99/년 + Google $25(1회). 그 외 클로드 작업분은 비용 0.
- 시간: 1~3단계(개발) 클로드가 수 세션. 4~6단계(계정·심사)는 애플 심사 대기 1~7일 변동.
- 구글이 애플보다 통과 쉬움 → **안드로이드(구글) 먼저 내고, iOS는 그 다음** 권장.

## 7. 다음 행동 (assignment)

**사장님이 먼저 결정할 1가지**: "지금 1단계(Capacitor 셋업)부터 클로드가 시작" vs "계정 가입($99 등)을 먼저 알아보고 시작". 
개발(1~3단계)은 계정 없이도 진행 가능하니, **클로드가 1단계부터 코드 작업을 먼저 시작**하고 계정은 심사 직전(5단계)에 만들어도 됩니다.

## 8. 미결 질문

- 결제를 정말 1차에서 빼도 되는지(현재 구독 매출 비중 확인 필요) — 빼도 웹 결제는 계속 살아있음.
- 앱 이름/아이콘을 현재 brand(keepioo·정책알리미) 그대로 갈지.
