# /mypage 탭 기반 레이아웃 재설계

- **작성일**: 2026-04-26
- **대상 파일**: `app/mypage/*`
- **목적**: 단일 컬럼으로 길게 늘어진 마이페이지를 탭 구조 + 데스크톱 2컬럼으로 재배치하여 공간 효율과 사용자 동선을 개선

## 1. 문제 정의

현재 `/mypage` 는 다음 4개 영역이 `max-w-[640px]` 단일 컬럼에 위→아래로 모두 펼쳐져 있다.

1. 프로필 폼 (이메일·나이·지역·시군구·직업·소득·가구·관심분야 = 8개 항목)
2. 동의 관리 (필수 2 + 선택 3 = 카드 5개)
3. 회원 탈퇴 (사유 라디오 6개 + 30일 안내 + 체크박스 + 버튼)

### 비효율 포인트

- **세로 길이 과다**: 데스크톱에서 스크롤이 길고 좌우 여백만 넓다.
- **자주 안 쓰는 영역도 항상 노출**: 동의 관리·탈퇴는 평소엔 거의 들어가지 않는데도 시각적 부담을 준다.
- **이메일 박스가 공간 차지**: 수정 불가인데 입력 박스 크기로 자리를 잡는다.
- **모든 항목이 평면**: "필수 vs 선택", "기본 vs 맞춤 추천"의 시각적 위계가 없다.
- **탈퇴 영역의 압박감**: 사유 라디오·체크박스가 평소에도 인라인으로 펼쳐져 있다.

## 2. 결정된 설계

### 2-1. 페이지 셸 — 상단 탭 3개

```
┌──────────────────────────────────────────────────┐
│  내 정보                          온보딩 다시 하기 │
│  📧 keeper0301@gmail.com                          │
│  기본 정보를 입력하면 맞춤추천과 알림이 더 정확해져요. │
├──────────────────────────────────────────────────┤
│  [ 프로필 ]  [ 동의 관리 ]  [ 계정 ]               │  ← 탭 (선택된 탭만 노출)
├──────────────────────────────────────────────────┤
│   (선택된 탭 본문)                                  │
└──────────────────────────────────────────────────┘
```

- 기본 노출 탭: **"프로필"**
- 컨테이너 너비: `max-w-[640px]` → `max-w-[920px]` (데스크톱 2컬럼 호흡 확보, 모바일은 풀폭으로 자연스럽게 떨어짐)
- 이메일은 헤더 보조 영역의 **작은 회색 텍스트**로 강등 (입력 박스 제거)
- 탭 라이브러리: **shadcn `Tabs`** (이미 같은 패키지의 `Dialog` 사용 중)

### 2-2. URL 동기화 + 앵커 호환

| 진입 경로 | 동작 |
|---|---|
| `/mypage` | 기본 "프로필" 탭 |
| `/mypage?tab=consents` | "동의 관리" 탭 |
| `/mypage?tab=account` | "계정" 탭 |
| `/mypage#consents` (legacy) | 마운트 시 1회 감지 → `?tab=consents` 로 자동 변환 + 스크롤 |

- 탭 전환 시 `router.replace(?tab=…)` 로 히스토리 오염 방지
- 클라이언트에서 `useSearchParams` 로 read·write
- 기존 `<section id="consents">` 의 직접 점프 호환을 위해 mount-once 처리

### 2-3. "프로필" 탭 — 두 그룹 + 데스크톱 2컬럼

논리적 분류:

| 그룹 | 항목 | 비고 |
|---|---|---|
| **기본 정보** | 나이대, 거주 지역(광역), 시·군·구, 직업 | 자주 바뀌지 않는 식별 정보 |
| **맞춤 추천 정보** | 소득 수준, 가구 상태, 관심 분야 | 선택 사항, 추천 정확도 향상용 |

레이아웃:

- 데스크톱 (`md` 이상): `grid-cols-2 gap-x-10` 2컬럼
- 모바일: 자연스럽게 1컬럼 stack
- 저장 버튼: 두 그룹 아래에 풀폭 1개 (현재 동작 유지)
- 그룹 헤더: `기본 정보 ─────` / `맞춤 추천 정보 ─────` 같은 라인 헤더

### 2-4. "동의 관리" 탭 — 필수 / 선택 그룹화

```
필수 동의 (철회 불가)
─────────
[🔒 개인정보처리방침]    동의 일자  [동의완료]
[🔒 이용약관]           동의 일자  [동의완료]

선택 동의 (언제든 끄고 켤 수 있어요)
─────────
[마케팅 정보 수신]              [ 토글 ]
[민감 토픽 분석]                [ 토글 ]
[카카오 알림톡 수신]            [ 토글 ]

ⓘ 필수 동의 철회는 회원 탈퇴를 통해 가능합니다.
```

- 카드 패딩 축소 (현재 ~110px 높이 → ~70px)
- 동의 일자는 우상단 작은 회색 텍스트
- 그룹 헤더로 "필수 / 선택" 시각적 분리

### 2-5. "계정" 탭 — 요약 카드 + 위험 영역

```
계정 관리

┌───────────────────────────────────┐
│ 가입 일자       2026년 4월 24일      │
│ 로그인 방식      구글 (이메일)        │
│ 알림톡 발송 수   N건 (이번 달)        │
└───────────────────────────────────┘

⚠️ 위험 영역
─────────────────────────────────────
[ 회원 탈퇴 ]
30일 유예 후 모든 데이터가 영구 삭제됩니다.
유예 기간 내 재가입 시 데이터 복구 가능.
                    [ 탈퇴 진행하기 ]
```

- 상단 **계정 요약 카드** 추가 (정보성, 빈 페이지 방지)
- 사유 라디오·체크박스를 **모달**(`Dialog`)로 이동
  - 평소에는 `[ 탈퇴 진행하기 ]` 버튼 1개만 노출
  - 클릭 시 모달에서 사유 6개 + 30일 안내 + 체크박스 + 최종 확인 한 번에
  - 기존 shadcn `Dialog` 인프라 재활용
- 탭 명칭은 **"계정"** (탈퇴 외 향후 확장 자리 확보)

### 2-6. 데이터 출처 (계정 요약)

| 필드 | 출처 | 추가 쿼리 |
|---|---|---|
| 가입 일자 | `supabase.auth.getUser()` 의 `user.created_at` | 없음 (이미 페이지에서 호출 중) |
| 로그인 방식 | `user.app_metadata.provider` | 없음 |
| 알림톡 발송 수 (이번 달) | `messaging_logs` 본인 row count | 페이지 SSR에 `count: 'exact'` 1건 추가 |

알림톡 카운트 쿼리는 기존 page.tsx의 `Promise.all` 묶음에 합류시켜 라운드트립 1회로 유지.

## 3. 파일 구조

```
app/mypage/
├── page.tsx              ← 수정 (서버: 데이터 페칭 + Tabs 셸 렌더)
├── tabs.tsx              ← 신규 (클라이언트: 탭 전환 + URL 동기화 + #anchor 처리)
├── profile-form.tsx      ← 수정 (2컬럼 그룹핑, 이메일 박스 제거)
├── consents-panel.tsx    ← 수정 (필수/선택 그룹 헤더 + 카드 컴팩트)
├── account-tab.tsx       ← 신규 (계정 요약 카드 + 탈퇴 진입 버튼)
└── withdraw-dialog.tsx   ← `withdraw-section.tsx` 리네임/리팩터
                              인라인 폼 → 모달 안의 폼으로 이동
```

각 파일 200줄 이하 (CLAUDE.md 규칙).

## 4. 영향 범위 (호환성)

- **외부 진입 링크**:
  - `/mypage#consents` → `?tab=consents` 로 자동 매핑 (앵커 호환 보장)
  - `/onboarding` → 그대로 (영향 없음)
- **권한·인증**: 변경 없음 (서버 컴포넌트 redirect 그대로)
- **데이터 모델**: 변경 없음 (UI 재배치만)
- **API**: 변경 없음 (`/api/consent`, `/api/account/delete` 그대로)
- **GA4 이벤트**: 변경 없음 (`ACCOUNT_DELETED` reason · has_detail 시그널 유지)
- **다른 페이지의 컨테이너 너비**: 영향 없음 (스코프 `app/mypage/page.tsx` 한정)

## 5. 비기능 요구사항

- **접근성**: shadcn Tabs 의 키보드 네비(←→) + ARIA 자동 처리
- **모바일**: 풀폭 1컬럼, 탭 가로 스크롤 없이 3개 한 줄 수용 (라벨 짧음)
- **성능**: SSR 쿼리는 동일 라운드트립 (Promise.all) 유지, 클라이언트 번들 영향은 shadcn Tabs 1개 추가 분량(~3KB gz)
- **저장 성공 UX**: 현재 "저장됐어요 ✓" 1.8초 토스트 동작 그대로 유지

## 6. 단계 (구현 순서 — writing-plans 단계에서 상세화)

1. shadcn `Tabs` 컴포넌트 설치 (의존성 추가)
2. `tabs.tsx` 신규 — URL 동기화 + #anchor 호환
3. `profile-form.tsx` — 2컬럼 그룹핑, 이메일 입력 박스 제거 (헤더 영역으로 이동)
4. `consents-panel.tsx` — 필수/선택 그룹 + 카드 패딩 축소
5. `account-tab.tsx` 신규 — 계정 요약 카드
6. `withdraw-section.tsx` → `withdraw-dialog.tsx` 리팩터 — 인라인 → 모달
7. `page.tsx` — 컨테이너 너비 확장 + 이메일 헤더 강등 + Tabs 셸 + 알림톡 카운트 쿼리 추가
8. 수동 QA: 탭 전환·URL 동기화·#consents 앵커·모바일 1컬럼·탈퇴 모달 플로우

## 7. 비목표 (이번 작업에서 안 하는 것)

- 비밀번호 변경·연동 계정 관리 등 "계정" 탭 신규 기능 (확장 자리만 확보)
- 동의 일자 표기 형식·동의 정책 자체 변경
- 회원 탈퇴 사유 옵션 변경 (GA4 dimension 안정성 유지)
- 프로필 옵션(나이·지역·소득·가구) 항목 자체 변경
- 다른 페이지(`/admin/*`, `/onboarding` 등) 레이아웃
