# keepioo 첫 외부 사용자 1명 만나기 — 도구 모음

생성: 2026-04-25
연관 자가진단: `2026-04-25-find-first-user.md`
상태: 사장님이 "다음 행동 4단계" 진행 시 옆에 두고 사용

---

## 사용 흐름

자가진단의 4단계 (지인 떠올리기 → 카톡 보내기 → 노트북 들고 만남 → 5분 관찰 → 한 마디 묻기) 각각에 도구 제공. 순서대로 사용.

---

## 1단계 — 자영업하는 지인 떠올리기

### 머릿속 체크리스트 (3분 안에)

다음 3 그룹에서 각 1명씩 떠올려보세요. 도합 **최소 3명**.

**A. 가족·친척**
- [ ] 가족 중 자영업하는 사람? (부모님·형제자매·배우자·사촌)
- [ ] 친척 중 식당·미용실·편의점·온라인 쇼핑몰 운영하는 사람?

**B. 친구**
- [ ] 친구 중 본인 사업하는 사람?
- [ ] 친구의 부모·형제 중 자영업자?

**C. 동네 가게 사장님 (자주 가는 곳)**
- [ ] 자주 가는 식당·카페·미용실·편의점 사장 중 안면 있는 분?
- [ ] 단골이라 카톡·전화번호 알고 있는 분?

### 떠오르지 않을 때

> "한 명도 안 떠오른다" 가 그 자체로 신호입니다.
> keepioo 의 페르소나 (40세 자영업자) 와 사장님 인맥의 거리감을 보여주는 것.
> 이 경우 → Approach B (자영업자 온라인 커뮤니티 모집) 으로 전환 권장.

## 2단계 — 카톡·전화 보내기 템플릿

### 톤 가이드
- 부담 0 — "구경만 해주세요" 강조 ("써보세요" 안 함)
- 시간 5분 명시 — 짧다는 안심
- 밥값 제안 — 가장 싼 인터뷰 비용 (점심·커피)
- "만들고 있는 거" 라고 표현 — 완성품 아니라 진행 중. 부담 ↓

### 템플릿 A — 가까운 가족·친척

```
형/누나/사촌, 잘 지내?
요즘 정부 지원금·복지 정보 한곳에 모아주는 사이트 만들고 있는데,
형/누나가 자영업이잖아. 5분만 둘러봐주면 안 될까?
밥은 내가 사 드릴게 ㅎㅎ 다음 주 언제 시간 돼?
```

### 템플릿 B — 친구

```
야 [이름]아, 부탁 하나 있는데
요즘 사이드로 사이트 하나 만들고 있어. 정부 지원금·소상공인 대출 정보 모아주는 거.
너 사업하니까 5분만 둘러봐주면 진짜 큰 도움 될 것 같은데 어때?
점심 사 줄게! 다음 주 언제 시간 돼?
```

### 템플릿 C — 동네 가게 사장님 (덜 친한 경우)

직접 가서 말하기 (방문):
```
"사장님, 안녕하세요. 매일 자주 오는 ○○○입니다.
실은 제가 정부 지원금·소상공인 대출 정보 모아주는 사이트를 만들고 있는데요,
사장님처럼 가게 운영하시는 분의 의견이 진짜 큰 도움이 됩니다.
혹시 5분만 시간 내주실 수 있을까요? 어느 시간이 한가하세요?"
```

### 메시지 보낸 뒤 체크
- [ ] 답장 24시간 내 옴 → 만남 약속 잡기
- [ ] 답장 없음 → 3일 후 한 번 더 ("저번에 보낸 거 잊지 마셔, 시간 언제 돼?")
- [ ] 거절 → 다음 후보 (1단계 리스트에서 다른 사람) 로

## 3단계 — 만남 전 준비

### 준비물 체크리스트
- [ ] 노트북 (충전 가득) — 와이파이 잘 되는 카페·식당 선택
- [ ] 종이 노트 + 펜 (관찰 메모용 — 키보드 두드리면 산만)
- [ ] 사장님 명함·연락처 (만남 후 후속 연락용)
- [ ] keepioo.com 미리 켜둔 브라우저 탭 — 만나서 바로 보여줄 수 있게

### 마인드셋 — 가장 중요
> **"이거 한 번 써봐 주세요" 한 마디만 하고 5분간 입 다물기.**
>
> 자기 사이트라고 자랑하고 싶은 본능 누르기. 표정·손동작이 진짜 정보입니다.
> "이거 어때요?" 같은 유도 질문 X. 그 사람 입에서 나오는 첫 한 마디만 진짜.

### 약속 잡기 시 주의
- 만남 시간: 점심 또는 그 사람 한가한 시간 (가게 사장님은 14~17시 추천)
- 장소: 그 사람 익숙한 곳 (단골 카페·자기 가게 한산한 시간)
- 본인이 노트북 들고 가서 그쪽이 보기 편하게

## 5단계 — 끝나기 전 마지막 한 마디

5분 후 사용자가 자연스럽게 손을 멈추면 한 마디만 묻기:

### 메인 질문

> **"이 사이트 친구한테 알려주실 거예요?"**

### 답에 따른 후속

**"예" 라고 하면:**
- "누구한테요?" → 어느 페르소나에 가치 있는지 정보
- "어떻게 알려주실 거예요? 카톡? 직접 말씀? 링크?" → distribution 채널 정보

**"아니요" 또는 "글쎄요" 면:**
- "어떤 부분이 부족해서요?" → 진짜 문제점 (정중하게 들어주기)
- 절대 변명·설득 X. 그냥 "아 그렇군요. 알려주셔서 감사해요" 만.

**모호하게 답하면 (가장 흔함):**
- "정확히 추천하실지 모르겠으면 — 어떤 점이 좋고 어떤 점이 부족했어요?"
- 좋은 점 1개 / 부족한 점 1개 까지 들어보기

### 연락처·후속 약속

- "혹시 며칠 후 한 번 더 연락드려도 될까요? 사이트 좀 고치고 다시 보여드릴게요" (재방문 의향 신호)
- 점심·커피값 결제 시 자연스럽게 마무리

## 만남 후 — 24시간 내 정리

### 즉시 (자리 일어선 직후 5분)

기억 남아있을 때 노트 옆에 다음 추가:
- 그 사람 핵심 한 마디 (가장 인상적이던 표현)
- 본인이 느낀 핵심 발견 1개 (한 줄로)

### 다음 날 (자고 일어난 뒤)

다음 docs 파일에 정리:

```markdown
# 첫 외부 사용자 만남 결과 — [날짜]

## 만난 사람
- 관계·나이·업종·지역

## 가장 강한 신호 3가지
1.
2.
3.

## "친구한테 알려줄 거예요?" 답 + 그 후속 답

## 가장 오래 머문 화면 + 추정 이유

## "이게 뭐예요?" 질문 5개 → keepioo 의 어느 단어가 모호한지

## 다음 한 달 keepioo 가 해야 할 일 (사장님 결정)
- 1순위:
- 2순위:
- 3순위:
```

이 정리를 Claude 와 공유하면 거기서부터 keepioo 의 다음 단계를 함께 결정.

## 자주 빠지는 함정

1. **자랑하기** — 사장님이 자기가 만든 거라고 설명하기 시작. 입 다물기.
2. **유도 질문** — "이 추천 카드 좋죠?" 같이 답을 정해놓고 묻기. "어때요?" 만 OK.
3. **친절한 거짓말 받아들이기** — "괜찮네~" 만 듣고 만족. 표정·손동작이 진짜 정보.
4. **혼자 너무 깊게 분석** — "그 사람이 그 페이지에서 멈춘 건 ~~ 때문일 거다" 추정. 다음 사람 만나서 검증.
5. **한 사람으로 결론** — 1명 만나고 keepioo 방향 확정. 최소 3명 만나야 패턴.

---

## 만남 횟수 가이드

| 만남 N번째 | 목적 | 결정 가능 |
|---|---|---|
| **1번째** | "사람이 어떻게 반응하나" 첫 인상 | 디자인 명백한 결함 |
| **2번째** | 첫 만남이 우연이 아닌지 검증 | 페르소나 가설 좁힘 |
| **3번째** | 패턴 확인 — 3명이 동일 단어 반복하면 진짜 | wedge·다음 한 달 우선순위 |
| **5번째** | 검증 완료 — keepioo 다음 분기 방향 결정 | 본격 빌드 재개 |

3명까지는 빌드 멈추고 만남 우선. 그 후 패턴 보고 빌드 재개.
